## Implementation Order & Prompts

### Prompt 1: CLI Interface Foundation
```markdown
## Task: Implement CLI Command System for OBLIQ Demo

### Context
Building on the completed Stage 1 WhatsApp clone, implement an in-composer CLI 
system that intercepts commands starting with `/` and provides autocomplete.

### Requirements

1. **Command Parser**
   - Intercept input starting with `/`
   - Parse command name and arguments
   - Support quoted strings: /view create "House Sale"

2. **Autocomplete UI**
   - Show dropdown when user types `/`
   - Filter commands as user types
   - Show command descriptions
   - Keyboard navigation (up/down/enter/escape)

3. **Command Registry**
   ```typescript
   interface Command {
     name: string;
     description: string;
     usage: string;
     handler: (args: string[]) => Promise<void>;
     suggestions?: () => string[];  // For argument autocomplete
   }
   ```

4. **Initial Commands to Implement**
   - `/help [topic]` - Show help
   - `/reset` - Reset demo state
   - `/theme dark|light` - Switch theme
   - `/settings` - Open settings modal

5. **UI Components**
   - CommandInput (enhanced message input)
   - CommandAutocomplete (dropdown)
   - CommandHelp (help modal)

6. **Styling**
   - Autocomplete: dark overlay, highlighted current selection
   - Command syntax highlighting in input
   - Success/error toast messages

### Files to Create/Modify
- `client/src/components/CommandInput.tsx`
- `client/src/components/CommandAutocomplete.tsx`
- `client/src/hooks/useCommands.ts`
- `client/src/commands/registry.ts`
- `client/src/commands/systemCommands.ts`

### Testing
- Type `/` shows all commands
- Type `/he` filters to `/help`
- Tab completes command
- Enter executes
- Escape closes autocomplete
```
